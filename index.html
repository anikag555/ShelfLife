ShelfLife: Predictive Novelty Purchase Timing

Introduction & Background

This project explores predictive analytics to identify when users are most likely to make novel purchases—items outside their regular staples. Our goal is to enhance retail strategies by predicting high-value shopping moments.

Problem & Motivation

Retailers often optimize for average behavior but miss potential profits from novelty behavior. We aim to forecast when deviations from standard purchase patterns are most likely to occur, using time-based features and user staples.

Team Structure

The team is composed of two members: one focused on data engineering and pipeline orchestration, the other on ML model development and dashboard visualization.

Technology Stack

  • Docker: For containerization
  • Databricks: For scalable ETL and ML training
  • Spark ML & scikit-learn: For model training
  • MLflow: Model tracking and deployment
  • AKS: API orchestration and deployment

Dataset & ETL

Data was sourced from Kaggle's Instacart Online Grocery Shopping Dataset. We cleaned and split the data in Databricks, using Delta Tables for storage and Spark transformations for ETL.

ML Model

We used a RandomForestClassifier to predict novelty purchase timing based on hour and weekday. MLflow was used to track performance, with an AUC of 0.78 and F1-score of 0.72.

REST APIs & Security

  • GET /user-staples: Returns staple items
  • POST /predict-novelty-time: Returns novelty probability

Security includes basic JWT validation, error logging, and retry logic using Python’s requests library.

Dashboard & Monitoring

Our Streamlit dashboard visualizes novelty probabilities, time trends, and system health. Alerts are triggered on latency spikes and low-confidence predictions.

GenAI Tool Reflection

OpenAI tools helped with debugging FastAPI, YAML scripting for AKS, and refining model insights. They improved efficiency but required careful verification for production use.
